1 Les Méthodes : Organiser son Code en Blocs Réutilisables

À mesure que les programmes grandissent, la méthode main devient surchargée et difficile à lire. Pour éviter cela et respecter le principe DRY (Don't Repeat Yourself), on divise le code en méthodes. Une méthode est un bloc de code nommé qui accomplit une tâche spécifique et peut être appelé à volonté.

Anatomie Détaillée d'une Méthode

Décortiquons la signature d'une méthode pour bien en comprendre chaque partie :

public static int sum(int a, int b)

  • - public : Le modificateur d'accès. Il définit qui peut appeler cette méthode. public signifie qu'elle est accessible depuis n'importe quelle autre classe de votre projet.

  • - static : Un mot-clé crucial que nous détaillerons plus bas. Pour l'instant, retenez qu'une méthode static appartient à la classe elle-même, et non à un objet spécifique. C'est pourquoi nous pouvons l'appeler directement depuis main, qui est aussi static.

  • - int : Le type de retour. C'est la promesse que la méthode fera un calcul et renverra une valeur de ce type. Si une méthode ne fait qu'effectuer une action sans renvoyer de valeur (comme afficher du texte), on utilise le mot-clé void.

  • - sum : Le nom de la méthode. Par convention, un nom de méthode commence par une minuscule et décrit une action (un verbe).

  • - (int a, int b) : La liste des paramètres. Ce sont les "ingrédients" dont la méthode a besoin pour fonctionner. a et b sont des variables locales à la méthode.

  • - { return a + b; } : Le corps de la méthode, où la logique est exécutée. L'instruction return met fin à la méthode et renvoie la valeur calculée.

 

Exemple 1 : Création de Méthodes Simples
xxxxxxxxxx
public class MethodExample {


    // Méthode qui prend deux entiers et retourne leur somme (type de retour : int)
    public static int sum(int a, int b) {
        return a + b;
    }


    // Méthode qui prend un nom et affiche un message (type de retour : void)
    public static void greet(String name) {
        System.out.println("Bonjour, " + name + "!");
    }


    public static void main(String[] args) {
        // Appel de la méthode sum et stockage du résultat
        int result = sum(10, 20);
        System.out.println("La somme de 10 et 20 est: " + result);


        // Appel de la méthode greet
        greet("Alice");
    }
}

2 La Surcharge de Méthodes (Method Overloading)

La surcharge de méthodes est une fonctionnalité très pratique de Java. Elle permet de définir plusieurs méthodes avec le même nom, à condition que leur signature de paramètres soit différente (soit par le nombre de paramètres, soit par leur type).

Analogie : Pensez aux boutons de votre télécommande. Vous pourriez avoir plusieurs boutons "Play", mais un pour le lecteur Blu-ray, un pour Netflix, un pour une clé USB. Vous appelez la même "action" (Play), mais le système sait laquelle exécuter en fonction du contexte (l'appareil actif).

Exemple 2 : Surcharge de la Méthode add
xxxxxxxxxx
public class MethodOverloading {


    // Version 1 : additionne deux entiers
    public static int add(int a, int b) {
        return a + b;
    }


    // Version 2 : surcharge avec trois entiers
    public static int add(int a, int b, int c) {
        return a + b + c;
    }


    // Version 3 : surcharge avec deux doubles
    public static double add(double a, double b) {
        return a + b;
    }


    public static void main(String[] args) {
        System.out.println("Addition de deux entiers: " + add(5, 10));
        System.out.println("Addition de trois entiers: " + add(1, 2, 3));
        System.out.println("Addition de deux doubles: " + add(4.5, 6.7));
    }
}

Le compilateur Java est assez intelligent pour appeler la bonne version de la méthode add en se basant sur les arguments que vous lui fournissez.

 

3 La Révolution : Penser en Termes de Classes et d'Objets

C'est le changement de mentalité le plus important en Java. Au lieu de penser en listes d'instructions, on pense en termes d'entités qui interagissent.

  • - Une Classe est un plan de construction, un moule.

    • - Analogie : Le plan d'architecte pour une maison. Il définit que toutes les maisons de ce type auront une cuisine, des chambres, et des portes, et qu'on pourra y "entrer" ou "allumer les lumières".

  • - Un Objet est une instance concrète créée à partir de ce plan.

    • - Analogie : La maison de Paul et la maison de Marie. Ce sont deux objets distincts, construits à partir du même plan. La maison de Paul peut avoir une cuisine peinte en bleu, celle de Marie en vert. Elles partagent la même structure, mais ont des états (des caractéristiques) différents.

Une classe regroupe deux types d'informations : les attributs (l'état) et les méthodes (le comportement). Créer un objet à partir d'une classe s'appelle l'instantiation, et se fait avec le mot-clé new.

Exemple 3 : La classe Voiture et sa création

1. Le plan de construction (Voiture.java)

xxxxxxxxxx
public class Voiture {
    // Attributs (état de chaque voiture)
    String marque;
    String modele;
    int vitesseActuelle = 0;


    // Méthodes d'instance (comportement)
    // Celles-ci ne sont PAS 'static' car elles agissent sur un objet spécifique
    public void accelerer(int augmentation) {
        this.vitesseActuelle += augmentation;
        System.out.println("La " + this.marque + " accélère. Vitesse : " + this.vitesseActuelle + " km/h.");
    }
}

2. Le programme principal qui utilise le plan (Garage.java)

xxxxxxxxxx
public class Garage {
    public static void main(String[] args) {
        // Instanciation : on crée deux objets Voiture distincts
        Voiture maVoiture = new Voiture();
        maVoiture.marque = "Renault";
        maVoiture.modele = "Clio";


        Voiture taVoiture = new Voiture();
        taVoiture.marque = "Peugeot";
        taVoiture.modele = "208";


        // Chaque objet a sa propre vie et son propre état
        maVoiture.accelerer(50);
        taVoiture.accelerer(30);
    }
}

 

4 Le Constructeur et le mot-clé this

Imaginez que vous êtes le directeur d'une usine de voitures. Chaque fois qu'une nouvelle voiture sort de la chaîne de montage, elle doit être prête à l'emploi. Elle a besoin de roues, d'un moteur et d'un volant. En programmation orientée objet, le constructeur est l'équivalent de cette chaîne de montage finale. Son rôle principal est d'initialiser un nouvel objet et de le mettre dans un état cohérent et utilisable dès sa création.

 

Qu'est-ce qu'un Constructeur ?

Un constructeur est une méthode spéciale qui est appelée automatiquement chaque fois que vous utilisez le mot-clé newpour créer une instance d'une classe. Contrairement aux autres méthodes, il n'a pas de type de retour (pas même void) et son nom doit être identique à celui de la classe.

Dans votre exemple, public Voiture(String marque, String modele) est le constructeur. Il prend deux paramètres (marque et modele) qui sont essentiels pour initialiser l'objet Voiture. Sans ces informations, la voiture ne serait qu'une coquille vide !

 

Le Rôle du mot-clé this

Lorsque vous donnez un nom à un paramètre de votre constructeur, il peut arriver que ce nom soit le même que celui d'un attribut de la classe. Dans l'exemple de la Voiture, le paramètre marque a le même nom que l'attribut marque.

C'est là que le mot-clé this entre en jeu. Il sert à lever l'ambiguïté.

  • - this.marque : Fait référence à l'attribut marque de l'objet en cours de création.

  • - marque (sans this) : Fait référence au paramètre marque passé au constructeur.

En utilisant this.marque = marque;, vous dites explicitement à Java : "Prends la valeur du paramètre marque et attribue-la à l'attribut marque de cet objet". C'est un moyen clair et non ambigu de lier les paramètres du constructeur aux attributs de l'objet.

 

L'Impact sur Votre Code

Sans constructeur, vous devriez initialiser chaque attribut manuellement, ce qui est fastidieux et source d'erreurs, surtout avec de nombreux attributs.

xxxxxxxxxx
// Version sans constructeur (fastidieuse et sujette aux erreurs)
Voiture maVoiture = new Voiture(); // Création de l'objet, mais ses attributs sont nuls.
maVoiture.marque = "Renault";
maVoiture.modele = "Clio";
maVoiture.vitesseActuelle = 0;

Avec le constructeur, la création de l'objet et son initialisation se font en une seule ligne, ce qui rend le code plus propre, plus lisible et moins propensoire aux erreurs.

xxxxxxxxxx
// Version avec constructeur (plus propre et sécurisée)
Voiture maVoiture = new Voiture("Renault", "Clio"); // Un seul appel

En résumé, le constructeur est la première méthode à s'exécuter lors de la création d'un objet. Il prépare l'objet à être utilisé, et le mot-clé this est l'outil qui vous permet de le faire de manière claire et structurée.

 

Exemple 4 : Classe Voiture avec un constructeur
xxxxxxxxxx
public class Voiture {
    String marque;
    String modele;
    int vitesseActuelle;


    // Le Constructeur
    public Voiture(String marque, String modele) {
        System.out.println("Construction d'une " + marque + " " + modele + "...");
        this.marque = marque; // 'this.marque' (attribut) reçoit la valeur du paramètre 'marque'
        this.modele = modele;
        this.vitesseActuelle = 0; // On initialise la vitesse à 0
    }


    public void accelerer(int augmentation) {
        this.vitesseActuelle += augmentation;
        System.out.println("La " + this.marque + " accélère. Vitesse : " + this.vitesseActuelle + " km/h.");
    }
}


// Nouvelle façon d'utiliser la classe dans Garage.java
public class Garage {
    public static void main(String[] args) {
        // On passe les valeurs directement lors de la création
        Voiture maVoiture = new Voiture("Renault", "Clio");
        Voiture taVoiture = new Voiture("Peugeot", "208");


        maVoiture.accelerer(50);
        taVoiture.accelerer(30);
    }
}

5 static vs. Instance : La Différence Fondamentale

Comprendre la différence entre les membres statiques et les membres d'instance est crucial en Java. Cette distinction est au cœur du modèle de programmation orientée objet. En substance, cela se résume à une question simple : la méthode ou la variable est-elle liée à un objet spécifique ou à la classe elle-même ?

 

Méthodes et Attributs d'Instance (Non-statiques)

Les membres d'instance (méthodes et attributs) appartiennent à un objet particulier. Chaque fois que vous créez un nouvel objet (une nouvelle "instance") de la classe, il a sa propre copie des attributs d'instance et peut utiliser les méthodes d'instance pour manipuler ces données.

  • - Analogie : Imaginez la classe Voiture. Chaque voiture que vous fabriquez est un objet distinct. Chaque voiture a sa propre couleur (un attribut d'instance) et sa propre méthode accelerer() (une méthode d'instance). L'action d'accélérer une voiture ne peut être effectuée que sur une voiture spécifique ; cela dépend de l'état de cet objet (sa vitesse actuelle).

  • - Utilisation : Pour accéder à un membre d'instance, vous devez d'abord créer un objet de la classe, puis utiliser l'opérateur point (.) sur cet objet.

    • Voiture maVoiture = new Voiture();

    • maVoiture.accelerer();

 

Méthodes et Attributs de Classe (static)

Les membres statiques (ou membres de classe) appartiennent à la classe elle-même, et non à un objet. Il n'existe qu'une seule copie de ces membres, partagée par toutes les instances de la classe. Ils sont souvent utilisés pour des utilitaires ou des valeurs qui n'ont pas de rapport avec l'état d'un objet en particulier.

  • - Analogie : Pensez à la classe Math de Java. Elle contient des méthodes comme Math.sqrt() (racine carrée) ou Math.abs() (valeur absolue). Pour calculer la racine carrée de 25, vous n'avez pas besoin de créer un objet "Math" (new Math()). La méthode sqrt() est un outil universel qui appartient à la classe Math. Elle n'a pas besoin de l'état d'un objet spécifique pour fonctionner.

  • - Utilisation : Les membres statiques sont appelés directement sur la classe en utilisant le nom de la classe, sans avoir besoin de créer un objet.

    • double resultat = Math.sqrt(25);

 

Exemple Pratique : Combiner les deux

L'exemple OutilDeCalcul illustre parfaitement cette combinaison.

  • - <nomCalculateur est un attribut d'instance. Chaque fois que vous créez un nouvel objet OutilDeCalcul, il a son propre nomCalculateur. La méthode afficherNom() est une méthode d'instance car elle utilise cet attribut d'instance (this.nomCalculateur). Elle dépend de l'état de l'objet.

  • - pi() est une méthode statique. La valeur de PI est une constante qui ne change jamais, peu importe l'objet OutilDeCalcul que vous utilisez. Il n'est donc pas logique qu'elle dépende d'un objet. Elle appartient à la classe elle-même, comme un outil de référence. C'est pourquoi on peut l'appeler directement via OutilDeCalcul.pi().

En bref, si un membre (variable ou méthode) a besoin de l'état d'un objet pour fonctionner, il doit être d'instance. S'il s'agit d'une fonction utilitaire ou d'une constante qui n'a pas besoin d'un objet pour exister, il doit être statique.
