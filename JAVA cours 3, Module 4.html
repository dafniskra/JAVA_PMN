12.1 Développement d'Applications de Bureau (GUI) avec JavaFX

 

  • Qu'est-ce que c'est ?

    C'est la création d'applications "classiques" qui s'exécutent sur un ordinateur (Windows, Mac, Linux) avec une interface graphique : des fenêtres, des menus, des boutons, des formulaires, etc.

  • Pourquoi est-ce utile ?

    Idéal pour des logiciels métier internes (outils de gestion, applications de comptabilité), des applications scientifiques, ou des logiciels de création.

  • La Technologie Clé : JavaFX

    JavaFX est le framework moderne et officiel pour construire des interfaces graphiques (GUI - Graphical User Interface) en Java. Il a remplacé l'ancien framework, Swing. JavaFX permet de créer des applications au design riche et réactif.

  • Les Concepts de Base de JavaFX :

    • Stage : La fenêtre principale de l'application.

    • Scene : Le contenu à l'intérieur de la fenêtre.

    • Nodes (ou Contrôles) : Les éléments graphiques comme Button, Label, TextField que vous placez dans la scène.

    • Mise en page (Layouts) : Des conteneurs comme VBox ou GridPane qui organisent automatiquement vos boutons et labels.

 

12.2 Développement Web Backend avec Spring Boot

 

  • Qu'est-ce que c'est ?

    C'est la construction du "cerveau" invisible qui fait fonctionner les sites et applications web. Le code backend s'exécute sur un serveur. Il gère la logique métier, traite les données, communique avec les bases de données et envoie les informations au "frontend" (ce que l'utilisateur voit dans son navigateur).

  • Pourquoi est-ce utile ?

    C'est le domaine le plus vaste et le plus porteur pour les développeurs Java. La quasi-totalité des grandes entreprises (banques, e-commerce, assurances...) utilisent Java pour leurs systèmes backend en raison de sa robustesse et de ses performances.

  • La Technologie Clé : Spring Framework & Spring Boot

    • Spring Framework : Une immense boîte à outils qui fournit des solutions pour presque tous les problèmes que l'on rencontre dans le développement d'applications d'entreprise.

    • Spring Boot : C'est la révolution qui a rendu Spring facile d'accès. Spring Boot est un projet qui simplifie radicalement la création d'applications web autonomes et prêtes pour la production. Il configure presque tout automatiquement pour vous. Aujourd'hui, pour le web en Java, on commence presque toujours par Spring Boot.

  • Les Concepts de Base de Spring Boot :

    • API REST : Créer des "endpoints" (des URLs) qui retournent des données (généralement au format JSON) pour que des applications frontend ou mobiles puissent les consommer.

    • Annotations : Spring Boot utilise massivement les annotations pour simplifier le code. Par exemple, @GetMapping("/utilisateurs") suffit à transformer une méthode Java en une page web accessible via une URL.

 

12.3 Développement Mobile avec Android

 

  • Qu'est-ce que c'est ?

    La création d'applications pour les smartphones et tablettes qui tournent sur le système d'exploitation Android de Google.

  • Pourquoi est-ce utile ?

    Le marché mobile est gigantesque, et Android en détient la plus grande part au niveau mondial. C'est un domaine créatif et en constante évolution.

  • Le Lien avec Java :

    Java a été pendant des années le langage officiel et principal pour le développement Android. Des milliards d'appareils font tourner du code Java.

    Note importante pour l'avenir : Aujourd'hui, Google promeut Kotlin comme son langage "préféré" pour le développement Android. La bonne nouvelle, c'est que Kotlin est un langage qui fonctionne sur la JVM et est 100% interopérable avec Java. Apprendre Java est donc une excellente porte d'entrée pour ensuite passer à Kotlin très facilement.

  • Les Outils : Le développement se fait principalement avec l'IDE Android Studio, fourni par Google.

 

12.4 Interaction avec les Bases de Données (JDBC & JPA)

 

  • Qu'est-ce que c'est ?

    Pratiquement toutes les applications ont besoin de stocker des données de manière permanente. Cette spécialisation consiste à apprendre comment une application Java se connecte à une base de données (comme MySQL, PostgreSQL, etc.) pour y lire, écrire, modifier et supprimer des données.

  • Pourquoi est-ce utile ?

    C'est une compétence transversale, absolument indispensable pour le développement backend, mais aussi utile dans de nombreux autres domaines.

  • Les Technologies Clés :

    1. JDBC (Java Database Connectivity) : L'API Fondamentale

      • C'est l'API de bas niveau, intégrée à Java, qui permet d'exécuter des requêtes SQL. Comprendre ses bases (Connection, Statement, ResultSet) est important pour savoir comment les choses fonctionnent "sous le capot".

    2. JPA (Java Persistence API) & Hibernate : L'Approche Moderne

      • Écrire du SQL à la main est répétitif et source d'erreurs. JPA est une spécification (un standard) pour le mapping objet-relationnel (ORM).

      • Analogie : L'ORM est un traducteur automatique. Vous manipulez vos objets Java (Voiture, Etudiant), et l'ORM (l'implémentation la plus célèbre étant Hibernate) se charge de générer et d'exécuter les requêtes SQL correspondantes (INSERT, UPDATE, SELECT...).

      • Dans le monde professionnel, on utilise quasiment toujours un ORM comme Hibernate (souvent via Spring Data JPA) pour interagir avec la base de données.
