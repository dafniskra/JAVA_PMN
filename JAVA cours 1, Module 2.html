1 Les Opérateurs : La Boîte à Outils de la Logique

Les opérateurs sont les symboles qui vous permettent de manipuler des variables et de formuler des questions.

Opérateurs Arithmétiques: Ce sont les bases du calcul.


Attention à la division entière : Si vous divisez deux entiers (int), le résultat sera un entier (la partie décimale est tronquée). Pour obtenir un résultat précis, au moins un des nombres doit être un double.

5.0 / 2.0 donne 2.5.

Raccourcis utiles :

  • - ++ (incrémenter) : x++ est équivalent à x = x + 1.

  • - -- (décrémenter) : x-- est équivalent à x = x - 1.

 

Opérateurs de Comparaison: Ils servent à poser des questions et le résultat est toujours un boolean (true ou false).


Erreur classique à éviter : Ne confondez jamais = (assignation : "mets cette valeur dans la variable") et == (comparaison : "cette valeur est-elle égale à celle-ci ?").

 

Opérateurs Logiques:Ils permettent de combiner plusieurs conditions.

  • - && (ET logique) : Vrai uniquement si toutes les conditions sont vraies.

    • Analogie : "Pour aller au cinéma, je dois avoir de l'argent ET avoir du temps libre."

  • - || (OU logique) : Vrai si au moins une des conditions est vraie.

    • Analogie : "Pour le dîner, je mangerai une pizza OU des pâtes."

  • - ! (NON logique) : Inverse la valeur d'un booléen.

    • Exemple : !true devient false.

xxxxxxxxxx
public class DemoOperateursLogiques {
    public static void main(String[] args) {
        int age = 20;
        boolean aLePermis = true;


        if (age >= 18 && aLePermis) {
            System.out.println("Vous pouvez conduire la voiture.");
        }
    }
}

2 Prise de Décision : Les Structures Conditionnelles

if, else if, else : Le Chemin à Plusieurs Voies

Cette structure permet à votre programme de suivre différents chemins en fonction des conditions.

if (Si) : Le premier chemin. Si la condition est vraie, on exécute ce bloc et on ignore le reste.

else if (Sinon si) : Un chemin alternatif. Si le if précédent était faux, on teste cette nouvelle condition. On peut en avoir plusieurs.

else (Sinon) : La voie par défaut. Si aucune des conditions précédentes n'était vraie, on exécute ce bloc.

xxxxxxxxxx
public class Conditions {
    public static void main(String[] args) {
        int score = 85;


        if (score >= 90) { // Est-ce que le score est >= 90 ? Non.
            System.out.println("Excellent ! Grade A.");
        } else if (score >= 80) { // OK, alors est-ce qu'il est >= 80 ? Oui !
            System.out.println("Très bien. Grade B."); // On exécute ce bloc et on s'arrête là.
        } else {
            System.out.println("Passable. Grade C ou inférieur.");
        }
    }
}

 

switch : L'Aiguillage

Quand vous avez une seule variable à comparer avec de nombreuses valeurs possibles, une longue suite de if-else if peut devenir lourde. Le switch est une alternative plus propre.

xxxxxxxxxx
public class DemoSwitch {
    public static void main(String[] args) {
        int jourDeLaSemaine = 4; // 1 = Lundi, 2 = Mardi, etc.
        String nomDuJour;


        switch (jourDeLaSemaine) {
            case 1:
                nomDuJour = "Lundi";
                break; // Le 'break' est crucial pour sortir du switch
            case 2:
                nomDuJour = "Mardi";
                break;
            case 3:
                nomDuJour = "Mercredi";
                break;
            case 4:
                nomDuJour = "Jeudi";
                break;
            case 5:
                nomDuJour = "Vendredi";
                break;
            default: // Le 'default' s'exécute si aucune autre 'case' ne correspond
                nomDuJour = "Weekend";
                break;
        }
        System.out.println("Aujourd'hui nous sommes " + nomDuJour);
    }
}

Résultat Attendu : Aujourd'hui nous sommes Jeudi

3 Répéter des Actions : Les Boucles

Les boucles sont un concept fondamental pour éviter la répétition de code (principe DRY : Don't Repeat Yourself).

La Boucle for : La Répétition Contrôlée

Utilisez for quand vous savez exactement combien de fois vous voulez répéter une action. Sa syntaxe se décompose en trois parties :

for ( initialisation ; condition ; miseàjour )

  1. Initialisation : Exécutée une seule fois au tout début (ex: int i = 0).

  2. Condition : Vérifiée avant chaque tour de boucle. Si c'est true, on continue.

  3. Mise à jour : Exécutée à la fin de chaque tour de boucle (ex: i++).

Java

xxxxxxxxxx
public class BoucleFor {
    public static void main(String[] args) {
        // Cette boucle va s'exécuter 5 fois (pour i de 0 à 4)
        for (int i = 0; i < 5; i++) {
            System.out.println("Le compteur est à : " + i);
        }
    }
}

 

La Boucle while : La Répétition Conditionnelle

Utilisez while quand vous voulez répéter une action tant qu' une condition reste vraie, sans savoir à l'avance combien de tours cela prendra.

Analogie : "Tant que la batterie du téléphone n'est pas à 100%, continuer à charger."

xxxxxxxxxx
public class BoucleWhile {
    public static void main(String[] args) {
        int munitions = 5;
        
        while (munitions > 0) {
            System.out.println("Pan ! Il reste " + (munitions - 1) + " munitions.");
            munitions--; // Très important : la condition doit pouvoir devenir fausse un jour !
        }
        System.out.println("Chargeur vide !");
    }
}

Résultat Attendu :

xxxxxxxxxx
Pan ! Il reste 4 munitions.
Pan ! Il reste 3 munitions.
Pan ! Il reste 2 munitions.
Pan ! Il reste 1 munitions.
Pan ! Il reste 0 munitions.
Chargeur vide !

 

La Boucle do-while : Exécuter au Moins une Fois

Cette boucle est une variante de while. Sa particularité est que la condition est vérifiée à la fin de chaque tour. Cela garantit que le code à l'intérieur de la boucle sera exécuté au moins une fois.

Analogie : Un menu dans un jeu. Vous voulez afficher les options au moins une fois avant de demander à l'utilisateur s'il veut quitter.

xxxxxxxxxx
import java.util.Scanner;


public class DemoDoWhile {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int choix;


        do {
            System.out.println("--- MENU ---");
            System.out.println("1. Jouer");
            System.out.println("2. Options");
            System.out.println("3. Quitter");
            System.out.print("Votre choix : ");
            choix = scanner.nextInt();
        } while (choix != 3); // On continue tant que l'utilisateur ne choisit pas de quitter
        
        System.out.println("Merci d'avoir joué !");
    }
}

4 Entrées et Sorties avec Scanner

Pour qu'un programme soit interactif, il doit pouvoir lire les informations que l'utilisateur saisit. C'est le rôle des entrées. De la même manière, il doit pouvoir communiquer un résultat à l'utilisateur, ce qui est le rôle des sorties. En Java, nous avons déjà vu la sortie avec System.out.println(). Pour les entrées, nous utilisons la classe Scanner.

La classe Scanner est un outil puissant de la bibliothèque standard de Java qui permet de lire des données de différentes sources, y compris le clavier de l'utilisateur (la console). Pour l'utiliser, il faut l'importer en début de fichier avec import java.util.Scanner;.

 

Créer et utiliser un objet Scanner

Pour commencer à lire les entrées, il faut d'abord créer un objet de la classe Scanner. On lui indique que la source des données est l'entrée standard du système (System.in), qui correspond au clavier.

Java

xxxxxxxxxx
Scanner scanner = new Scanner(System.in);

Une fois que l'objet scanner est créé, vous pouvez utiliser ses méthodes pour lire différents types de données :

  • scanner.nextLine() : Lit une ligne de texte complète, jusqu'à ce que l'utilisateur appuie sur la touche Entrée. C'est la méthode idéale pour récupérer une chaîne de caractères (String).

  • scanner.nextInt() : Lit un nombre entier (int). Si l'utilisateur tape du texte à la place d'un nombre, le programme générera une erreur.

  • scanner.nextDouble() : Lit un nombre décimal (double).

 

Les Bonnes Pratiques avec Scanner

  1. Guider l'utilisateur : Avant de demander une entrée, il est essentiel d'afficher un message clair avec System.out.print(). Cela évite que l'utilisateur ne se demande ce qu'il doit faire.

  2. Fermer le Scanner : Une fois que vous n'avez plus besoin de l'objet Scanner, vous devez le fermer en appelant la méthode scanner.close(). Cela libère les ressources système et empêche les fuites de ressources. C'est une bonne habitude à prendre pour écrire du code propre et efficace.

 

Exemples Détaillés

Exemple 6 : Récupérer des informations de l'utilisateur

Java

xxxxxxxxxx
import java.util.Scanner;


public class UserInput {
    public static void main(String[] args) {
        // 1. Création de l'objet Scanner pour lire les entrées clavier
        Scanner scanner = new Scanner(System.in);


        // 2. Demander et lire le nom (une chaîne de caractères)
        System.out.print("Entrez votre nom: ");
        String name = scanner.nextLine(); // Lit la ligne entière saisie par l'utilisateur


        // 3. Demander et lire l'âge (un nombre entier)
        System.out.print("Entrez votre âge: ");
        int age = scanner.nextInt(); // Lit le prochain entier saisi


        // 4. Afficher les informations récupérées
        System.out.println("Bonjour " + name + ", vous avez " + age + " ans.");
        
        // 5. Fermeture du scanner
        scanner.close(); 
    }
}

Exemple 7 : Effectuer des calculs avec des entrées utilisateur

Java

xxxxxxxxxx
import java.util.Scanner;


public class BasicMath {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);


        System.out.print("Entrez un premier nombre entier: ");
        int number1 = scanner.nextInt();


        System.out.print("Entrez un second nombre entier: ");
        int number2 = scanner.nextInt();


        // Effectuer les opérations mathématiques
        int sum = number1 + number2;
        int product = number1 * number2;


        // Afficher les résultats
        System.out.println("La somme de " + number1 + " et " + number2 + " est: " + sum);
        System.out.println("Le produit de " + number1 + " et " + number2 + " est: " + product);


        scanner.close();
    }
}

Ces exemples montrent comment la classe Scanner rend vos programmes dynamiques et interactifs, permettant à l'utilisateur d'interagir directement avec votre code.

5 Conversions de Types

En Java, chaque variable a un type de données spécifique (comme int, double, String, etc.). Parfois, vous avez besoin de convertir une valeur d'un type à un autre. Ce processus s'appelle le casting. Il existe deux types de conversions : implicite et explicite.

Conversion Implicite (Widening Conversion)

La conversion implicite, ou "élargissement" (widening), se produit automatiquement. Elle a lieu lorsque vous convertissez un type de données plus petit vers un type plus grand, sans risque de perte d'informations. Pensez à verser de l'eau d'un petit verre dans un grand verre : il n'y a aucun débordement.

Quand cela se produit-il ? Quand vous passez d'un type à une "plage de valeurs" plus large, par exemple :

  • byte -> short -> int -> long

  • int -> float -> double

Exemple 8 : Conversion implicite

xxxxxxxxxx
public class TypeCasting {
    public static void main(String[] args) {
        // int peut tenir dans un double sans problème, la conversion est automatique.
        int intValue = 100;
        double doubleValue = intValue;  
        System.out.println("Conversion implicite de int en double: " + doubleValue); // Affiche 100.0
    }
}

 

Conversion Explicite (Narrowing Conversion)

La conversion explicite, ou "rétrécissement" (narrowing), n'est pas automatique. Le compilateur vous oblige à le faire manuellement en utilisant des parenthèses (type). Cela est nécessaire lorsque vous convertissez un type de données plus grand vers un type plus petit. Il y a un risque de perte de données. Pensez à verser de l'eau d'un grand verre dans un petit : une partie de l'eau risque de déborder et d'être perdue.

Quand cela se produit-il ? Quand vous passez d'un type à une "plage de valeurs" plus étroite, par exemple :

  • double -> float

  • double -> int

  • long -> int

Exemple 8 (suite) : Conversion explicite

xxxxxxxxxx
public class TypeCasting {
    public static void main(String[] args) {
        double anotherDouble = 9.78;
        // On force la conversion en int. La partie décimale (.78) est perdue.
        int narrowedInt = (int) anotherDouble;
        System.out.println("Conversion explicite de double en int: " + narrowedInt); // Affiche 9
    }
}

 

Conversion entre String et Types Numériques

Un cas de conversion très fréquent est de passer d'une chaîne de caractères (String) à un nombre et vice-versa. C'est particulièrement utile lorsque vous lisez une entrée de l'utilisateur (qui est toujours lue en tant que String par Scanner.nextLine()) et que vous devez la traiter comme un nombre.

Pour cela, vous utilisez les classes enveloppes (wrapper classes) comme Integer ou Double.

  • Integer.parseInt(votreString) : Convertit une chaîne de caractères en un int.

  • Double.parseDouble(votreString) : Convertit une chaîne de caractères en un double.

Exemple 9 : Conversion de chaînes en types numériques

xxxxxxxxxx
import java.util.Scanner;


public class StringToNumber {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);


        System.out.print("Entrez votre âge (en texte): ");
        String inputAge = scanner.nextLine();
        // Le texte "25" est converti en le nombre entier 25.
        int age = Integer.parseInt(inputAge);
        System.out.println("Vous avez " + age + " ans.");
        
        scanner.close();
    }
}

Si l'utilisateur entre un texte qui ne peut pas être converti en nombre (comme "trente"), votre programme se terminera avec une erreur. C'est un aspect important de la gestion des erreurs que vous apprendrez plus tard.

 

Résumé des Conversions de Types Primitifs

Exemple 10 : Conversions entre types primitifs

 

xxxxxxxxxx
public class PrimitiveConversion {
    public static void main(String[] args) {
        // Conversion de int à long : implicite, car long a une plus grande capacité que int.
        int smallNumber = 100;
        long largeNumber = smallNumber;
        System.out.println("int converti en long: " + largeNumber); // Affiche 100


        // Conversion de double à float : explicite, car double a une plus grande précision.
        // Il y a un risque de perte de précision.
        double doubleValue = 10.5;
        float floatValue = (float) doubleValue;
        System.out.println("double converti en float: " + floatValue); // Affiche 10.5
    }
}

Comprendre ces règles est essentiel pour éviter les erreurs de compilation et les problèmes de logique dans vos programmes.
