Quiz : Java - POO et Bibliothèque Standard (Modules 5-8)

Module 5 : La Programmation Orientée Objet (POO)

  1. Quel est l'objectif principal de l'encapsulation en POO ?

    • A) Permettre à une classe d'hériter des méthodes d'une autre classe.

    • B) Créer plusieurs formes d'une même méthode pour différents types d'objets.

    • C) Regrouper les données et les méthodes qui les manipulent, tout en protégeant les données d'un accès direct non contrôlé.

    • D) Rendre toutes les méthodes d'une classe publiques pour un accès facile.


    • Réponse correcte : C) Regrouper les données et les méthodes qui les manipulent, tout en protégeant les données d'un accès direct non contrôlé.

    • Raisonnement : L'encapsulation vise à protéger l'intégrité de l'état d'un objet en rendant ses attributs private et en ne fournissant qu'un accès contrôlé via des méthodes publiques (getters/setters).

  2. Dans l'héritage, comment une classe fille peut-elle appeler le constructeur de sa classe mère ?

    • A) En utilisant le mot-clé this().

    • B) En utilisant le mot-clé super().

    • C) En utilisant le mot-clé parent().

    • D) Le constructeur de la classe mère est appelé automatiquement sans action requise.


    • Réponse correcte : B) En utilisant le mot-clé super().

    • Raisonnement : Le mot-clé super fait référence à la classe mère. L'appel super(...) dans un constructeur de la classe fille invoque le constructeur correspondant de la classe mère pour initialiser les attributs hérités.

  3. Considérez le code Animal monAnimal = new Chien();. Quelle est la manifestation la plus claire du polymorphisme ici ?

    • A) On ne peut pas assigner un objet Chien à une variable Animal.

    • B) L'objet Chien peut maintenant utiliser des méthodes de la classe Animal.

    • C) Une variable de type Animal peut contenir une référence vers un objet d'une de ses sous-classes (Chien).

    • D) La classe Animal doit être déclarée final.


    • Réponse correcte : C) Une variable de type Animal peut contenir une référence vers un objet d'une de ses sous-classes (Chien).

    • Raisonnement : Le polymorphisme signifie "plusieurs formes". Il permet de traiter un objet (Chien) comme une instance de son type parent (Animal), ce qui rend le code plus flexible et générique.

  4. Quelle est la différence fondamentale entre une interface et une classe abstraite ?

    • A) Une classe ne peut implémenter qu'une seule interface, mais hériter de plusieurs classes abstraites.

    • B) Une interface ne peut contenir que des signatures de méthodes (un contrat), alors qu'une classe abstraite peut aussi contenir du code d'implémentation partagé.

    • C) Les classes abstraites sont utilisées pour l'encapsulation, et les interfaces pour le polymorphisme.

    • D) Il n'y a pas de différence majeure, ce sont deux termes pour le même concept.


    • Réponse correcte : B) Une interface ne peut contenir que des signatures de méthodes (un contrat), alors qu'une classe abstraite peut aussi contenir du code d'implémentation partagé.

    • Raisonnement : Une interface définit une capacité (ce qu'un objet peut faire), tandis qu'une classe abstraite définit une identité (ce qu'un objet est) et peut fournir une implémentation de base commune à toutes ses sous-classes.

 

Module 6 : La Gestion des Exceptions

 

  1. Dans un bloc try-catch-finally, quand le code du bloc finally est-il exécuté ?

    • A) Uniquement si une exception est attrapée par le bloc catch.

    • B) Uniquement si aucune exception ne se produit dans le bloc try.

    • C) Jamais, c'est une syntaxe optionnelle rarement utilisée.

    • D) Toujours, qu'une exception se soit produite ou non.


    • Réponse correcte : D) Toujours, qu'une exception se soit produite ou non.

    • Raisonnement : Le bloc finally est conçu pour contenir du code de "nettoyage" (comme fermer un fichier ou une connexion) qui doit impérativement être exécuté, quel que soit le résultat du bloc try.

  2. Quel mot-clé est utilisé pour lancer manuellement une exception dans le code ?

    • A) throws

    • B) catch

    • C) throw

    • D) new


    • Réponse correcte : C) throw.

    • Raisonnement : L'instruction throw new MyException(); est utilisée pour déclencher activement une exception lorsque votre code rencontre une condition d'erreur logique. throws, quant à lui, est utilisé dans la signature d'une méthode pour déclarer qu'elle peut lancer une exception.

  3. Si un utilisateur entre le texte "douze" lorsque le programme attend un entier avec scanner.nextInt(), quelle exception sera typiquement levée ?

    • A) NumberFormatException

    • B) IllegalArgumentException

    • C) InputMismatchException

    • D) IOException


    • Réponse correcte : C) InputMismatchException.

    • Raisonnement : Cette exception spécifique à la classe Scanner est levée lorsque le type de l'entrée ne correspond pas au type attendu par la méthode (nextInt, nextDouble, etc.).

  4. Comment crée-t-on une exception personnalisée en Java ?

    • A) En créant une classe qui implémente l'interface Throwable.

    • B) En créant une classe qui hérite de la classe Error.

    • C) En créant une classe qui hérite de la classe Exception (ou d'une de ses sous-classes).

    • D) En utilisant l'annotation @Exception.


    • Réponse correcte : C) En créant une classe qui hérite de la classe Exception (ou d'une de ses sous-classes).

    • Raisonnement : C'est le mécanisme standard pour créer des types d'erreurs spécifiques à la logique de votre application, comme InvalidAgeException ou InsufficientFundsException.

 

Module 7 : Bibliothèque Standard et Collections

 

  1. Quel est le principal avantage d'une ArrayList par rapport à un tableau standard (String[]) ?

    • A) Elle est plus rapide pour accéder aux éléments.

    • B) Elle peut contenir plusieurs types de données différents en même temps.

    • C) Sa taille est dynamique et peut augmenter ou diminuer selon les besoins.

    • D) Elle utilise moins de mémoire.


    • Réponse correcte : C) Sa taille est dynamique et peut augmenter ou diminuer selon les besoins.

    • Raisonnement : Contrairement aux tableaux dont la taille est fixe à la création, une ArrayList gère automatiquement le redimensionnement, ce qui la rend beaucoup plus flexible.

  2. Dans une HashMap<String, Integer>, comment récupère-t-on la valeur associée à la clé "score" ?

    • A) map.get("score")

    • B) map.find("score")

    • C) map["score"]

    • D) map.fetch("score")


    • Réponse correcte : A) map.get("score").

    • Raisonnement : La méthode get(key) est la manière standard de récupérer la valeur associée à une clé dans une HashMap. Si la clé n'existe pas, elle retourne null.

  3. Quelle est la caractéristique principale d'une HashSet ?

    • A) Elle maintient les éléments dans l'ordre où ils ont été insérés.

    • B) Elle ne contient que des éléments uniques et ignore les doublons.

    • C) Elle est optimisée pour stocker des paires clé-valeur.

    • D) Elle permet un accès aux éléments par un index numérique.


    • Réponse correcte : B) Elle ne contient que des éléments uniques et ignore les doublons.

    • Raisonnement : L'objectif d'un HashSet est de garantir l'unicité des éléments qu'il contient. Toute tentative d'ajout d'un élément déjà présent est simplement ignorée.

  4. Quand devriez-vous préférer utiliser une classe abstraite plutôt qu'une interface ?

    • A) Lorsque vous voulez qu'une classe puisse implémenter plusieurs comportements.

    • B) Lorsque vous voulez simplement définir un contrat sans aucune implémentation.

    • C) Lorsque plusieurs classes partagent une identité commune et que vous voulez partager du code (des méthodes concrètes) entre elles.

    • D) Toujours, car les classes abstraites sont plus modernes que les interfaces.


    • Réponse correcte : C) Lorsque plusieurs classes partagent une identité commune et que vous voulez partager du code (des méthodes concrètes) entre elles.

    • Raisonnement : La classe abstraite est idéale pour modéliser une relation "est-un" (Chien est un Animal) et factoriser du code commun dans les méthodes concrètes, tout en forçant l'implémentation de méthodes abstraites.

 

Module 8 : Fichiers et Entrées/Sorties

 

  1. Quelle est la manière moderne et simple recommandée pour lire tout le contenu d'un petit fichier texte en une seule fois ?

    • A) new FileInputStream("fichier.txt")

    • B) new Scanner(new File("fichier.txt"))

    • C) Files.readString(Path.of("fichier.txt"))

    • D) new BufferedReader(new FileReader("fichier.txt"))


    • Réponse correcte : C) Files.readString(Path.of("fichier.txt")).

    • Raisonnement : La classe Files de l'API java.nio (New I/O) fournit des méthodes statiques très pratiques comme readString et writeString pour des opérations de fichier simples et concises.

  2. Quel est le principal avantage de la structure try-with-resources lors de la manipulation de fichiers ?

    • A) Elle rend le code plus rapide.

    • B) Elle garantit que le flux (ex: BufferedReader) sera automatiquement fermé à la fin du bloc, même en cas d'erreur.

    • C) Elle permet d'écrire et de lire un fichier en même temps.

    • D) Elle supprime automatiquement le fichier après utilisation.


    • Réponse correcte : B) Elle garantit que le flux (ex: BufferedReader) sera automatiquement fermé à la fin du bloc, même en cas d'erreur.

    • Raisonnement : C'est une bonne pratique essentielle qui évite les fuites de ressources en s'assurant que la méthode close() de la ressource est toujours appelée, ce qui rend le bloc finally superflu pour cette tâche.

  3. Comment faut-il initialiser un FileWriter pour ajouter du texte à la fin d'un fichier existant sans l'écraser ?

    • A) new FileWriter("fichier.txt").append()

    • B) new FileWriter("fichier.txt", true)

    • C) new FileWriter("fichier.txt", "append")

    • D) new FileWriter("fichier.txt").mode(APPEND)


    • Réponse correcte : B) new FileWriter("fichier.txt", true).

    • Raisonnement : Le constructeur de FileWriter accepte un second argument booléen. Si ce dernier est true, le flux est ouvert en mode "ajout" (append), et les nouvelles données sont écrites à la fin du fichier.

  4. Pourquoi la plupart des méthodes de manipulation de fichiers déclarent-elles throws IOException?

    • A) Parce que IOException est une erreur de logique que le programmeur doit corriger.

    • B) Parce que les opérations sur les fichiers sont intrinsèquement risquées (fichier non trouvé, disque plein, pas de permissions) et Java force le développeur à gérer ces erreurs potentielles.

    • C) Pour obliger le programmeur à utiliser la nouvelle API java.nio.

    • D) C'est une convention qui n'a pas d'impact technique.


    • Réponse correcte : B) Parce que les opérations sur les fichiers sont intrinsèquement risquées (fichier non trouvé, disque plein, pas de permissions) et Java force le développeur à gérer ces erreurs potentielles.

    • Raisonnement : IOException est une "exception vérifiée" (checked exception). Le compilateur Java impose sa gestion via un bloc try-catch ou en la propageant, ce qui rend les programmes qui manipulent des fichiers plus robustes et fiables.
